You are acting as a senior QA engineer and backend tester.

Your task is to perform a COMPLETE end-to-end validation of the application, covering authentication, Stripe payments, database integrity, and UI consistency.

Follow these steps STRICTLY and report results clearly for each step.

---

## 1. AUTHENTICATION TESTING

Test both registration and login flows:

1. Register a new user
   - Use a test email and password
   - Confirm:
     - API returns success response
     - User is actually created in the database (verify via DB query)

2. Login with the same credentials
   - Confirm:
     - Authentication succeeds
     - Session/token is properly issued
     - No login loop or immediate logout occurs

3. Validate:
   - Session persistence (refresh or re-request)
   - No authentication errors in logs

---

## 2. DATABASE VERIFICATION (AUTH)

- Query the database directly
- Confirm:
  - New user exists
  - Fields are correctly populated (id, email, timestamps, etc.)
  - No duplicate or malformed entries

---

## 3. STRIPE CHECKOUT TEST

Perform a full payment flow:

1. Create or use a Stripe test customer (if required)
2. Trigger checkout via `/api/billing/checkout`
3. Complete payment using Stripe test card (4242 4242 4242 4242)

Validate:
- Checkout session is successfully created
- Payment completes without errors

---

## 4. WEBHOOK VALIDATION

Monitor webhook endpoint: `/api/stripe/webhook`

Confirm the following events are received and handled correctly:

- checkout.session.completed
- payment_intent.succeeded
- invoice.payment_succeeded
- customer.subscription.created (if applicable)

For each event:
- Ensure HTTP 200 response
- Ensure no signature verification errors
- Ensure correct event handling logic executes

---

## 5. DATABASE VERIFICATION (PAYMENTS)

After successful payment:

- Verify database updates:
  - Payment record created
  - Subscription (if applicable) created/updated
  - User account reflects purchase (tier, credits, etc.)

- Check:
  - No duplicate entries
  - Correct mapping between Stripe customer and internal user

---

## 6. FAILURE SCENARIOS

Trigger and validate:

- Failed payment (use Stripe test failure card)
- Confirm:
  - Proper error handling
  - No incorrect database updates
  - System remains stable

---

## 7. UI VALIDATION

Check frontend behavior:

- After login:
  - User remains logged in
  - UI reflects authenticated state

- After successful payment:
  - UI updates correctly (subscription/tier/credits)
  - No stale or incorrect state

---

## 8. LOGS & ERROR HANDLING

- Inspect server logs throughout testing
- Identify:
  - Errors
  - Warnings
  - Silent failures

---

## 9. FINAL REPORT

Provide a structured report:

- ✅ What is working correctly
- ❌ What is broken or inconsistent
- ⚠️ Any risks before production
- 🔧 Suggested fixes (if issues found)

Be precise. Do NOT assume success — verify everything using actual responses, logs, and database state.
